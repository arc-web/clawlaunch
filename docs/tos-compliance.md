# Anthropic TOS Compliance Analysis

*Last updated: March 2026*

---

## Summary

ClawLaunch is compliant with Anthropic's Consumer Terms of Service. It uses the official Claude Code CLI binary with an OAuth token generated by Anthropic's own `claude setup-token` command. No tokens are extracted, shared, or used in third-party tools.

---

## Relevant TOS Provisions

### Section 3.7 - Automated Access

The Consumer Terms of Service state:

> "Except when you are accessing our Services via an Anthropic API Key or where we otherwise explicitly permit it, [you agree not] to access the Services through automated or non-human means, whether through a bot, script, or otherwise."

This creates **two exceptions** to the prohibition on automated access:

1. **Via an Anthropic API Key** - covered by Commercial Terms of Service
2. **Where Anthropic "otherwise explicitly permits it"** - Claude Code CLI is that explicit permission

### February 2026 Clarification

Anthropic added:

> "Using OAuth tokens obtained through Claude Free, Pro, or Max accounts in any other product, tool, or service - including the Agent SDK - is not permitted and constitutes a violation of the Consumer Terms of Service."

Key phrase: **"any other product, tool, or service."** Claude Code CLI is not "another" product - it is Anthropic's own product.

---

## Why ClawLaunch Is Compliant

### 1. We use the official Claude Code CLI

ClawLaunch invokes the `claude` binary - Anthropic's own product, installed via `npm install -g @anthropic-ai/claude-code`. We do not:

- Extract tokens for use in other HTTP clients
- Build custom API wrappers around the OAuth token
- Use any binary other than the official Claude Code CLI

### 2. We use Anthropic's own authentication mechanism

The `claude setup-token` command is built into the Claude Code CLI by Anthropic. It generates a long-lived OAuth token specifically designed for:

- CI/CD pipelines
- Headless server environments
- Remote development machines

Running this command and using the resulting token with the Claude CLI is the documented, intended use case.

### 3. Our architecture matches Claude Code Channels

In March 2026, Anthropic launched **Claude Code Channels** - an official feature that:

- Runs the Claude CLI as a persistent Discord/Telegram bot
- Requires a Pro/Max subscription (API keys are explicitly *not* supported for Channels)
- Runs on user infrastructure (VPS, home server, etc.)

ClawLaunch's architecture is functionally identical: a persistent process on a VPS that invokes the Claude CLI for Discord interactions. The only difference is the orchestration layer (OpenClaw vs. Anthropic's built-in channel plugins).

### 4. The shim does not circumvent protections

The Claude CLI shim at `/data/.agents/bin/claude` does one thing: `unset ANTHROPIC_API_KEY` before calling the real `claude` binary. This *prevents* API key billing and *forces* OAuth token authentication. It's a billing routing choice, not a security bypass.

## What Would Be Non-Compliant

For clarity, here is what ClawLaunch explicitly avoids:

| Action | Why It's Prohibited | ClawLaunch Status |
|--------|-------------------|-------------------|
| Extracting OAuth tokens for third-party tools | TOS Section 3.7 + Feb 2026 clarification | Not done |
| Using subscription tokens with Agent SDK | Explicitly prohibited by Anthropic | Not done |
| Token reselling or sharing | TOS Section 3.7 | Not done |
| Using tokens in Cursor, OpenCode, Roo Code | Feb 2026 enforcement action | Not done |
| Bypassing rate limits | TOS general provisions | Not done |
| Reverse engineering the Claude CLI | TOS Section 3.7 | Not done |

## Enforcement Timeline

### January 9, 2026 - Server-Side Blocks

Anthropic deployed server-side blocks that prevented third-party tools from authenticating with subscription OAuth tokens. Affected tools:

- OpenCode (56,000+ GitHub stars) - later removed Claude support citing "Anthropic legal requests"
- Various Claude-compatible IDE extensions
- Third-party API wrappers

### January-February 2026 - Account Bans

Some Max subscribers reported bans. Anthropic later reversed erroneous bans. Pattern: bans targeted accounts using tokens in non-Anthropic clients, not accounts using Claude Code normally.

### February 2026 - Official Clarification

Anthropic engineer Thariq stated on X:

> "We haven't changed anything here. It's not against terms of service to have multiple MAX accounts."

> "Third-party harnesses using Claude subscriptions create problems for users and are prohibited by our Terms of Service."

Enforcement targets: **token resellers** and **third-party harnesses**. Not subscribers using Claude Code.

### March 20, 2026 - Claude Code Channels Launch

Anthropic shipped Claude Code Channels as a research preview. This feature:

- Officially supports running Claude as a Discord/Telegram bot
- Requires Pro/Max subscription (not API keys)
- Validates the "always-on agent on a VPS" use pattern

## Risk Assessment

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| Anthropic restricts CLI usage on VPS | Very low | High | Would break their own Channels feature and CI/CD docs |
| Rate limits tightened for Max plan | Medium | Medium | Fall back to API keys for overflow |
| `setup-token` deprecated | Low | Low | Switch to standard `claude login` with shorter refresh cycles |
| TOS changes prohibit all automated subscription use | Very low | High | Migrate to API keys; cost increase |
